# 部署说明

## 环境划分

- `dev`：开发人员本地调试与预览。
- `prod`：正式生产环境，建议部署至中国香港源站并按访问效果决定是否追加 CDN。

## 最小部署流程

1. 本地执行 `npm install`。
2. 本地执行 `npm run dev` 或 `npm run build`。
3. 提交代码并创建合并请求。
4. 由 GitHub Actions 执行构建与检查。
5. 合并主分支后，将 `dist/` 发布到正式源站。
6. 执行 CDN 刷新或关键页面预热。

## 回滚建议

1. 保留最近数个 `dist` 构建版本。
2. 每次生产发布前记录当前版本号与发布时间。
3. 若新版本异常，直接回切到上一个静态产物目录。
