# 相弈象棋像素级复刻：技术规格

版本：v0.3  
日期：2026-04-28

## 1. 项目目标

在 `xiangqi_auto` 内复刻 `https://www.zh.xiangqi.com/` 及其全部可见子页面，要求：

- 首页像素级复刻。
- 全部子页面逐步复刻。
- 简中、繁中、英语、Tiếng Việt 均做实际页面。
- 繁中、英语、Tiếng Việt 页面按简体中文界面的同等标准复刻：页面结构、视觉细节、图片图标、SEO 内容、文章/FAQ/帮助交互、导航行为、移动端适配都必须完整实现。
- 保留 SEO 长内容。
- 教学文章和 FAQ 的交互效果必须实现。
- 图片、CSS、JS 等静态资源本地化。
- 允许指定业务外链：`play.xiangqi.com`、真实下载链接、社交媒体链接。

## 2. 技术栈

| 项目 | 规格 |
| --- | --- |
| 框架 | Astro |
| 输出 | 静态站点 |
| 样式 | 本地 CSS，按源站视觉重写 |
| 脚本 | 本地原生 JavaScript 或 Astro island；禁止依赖源站运行时 |
| 图片 | 本地 `public` 目录 |
| 字体 | 优先系统字体；如源站使用特殊字体，需下载并本地托管 |
| 数据 | JSON / Markdown / Astro 内容集合 |

## 3. 外链和本地化规则

### 允许保留的外链

- 在线对弈：`https://play.xiangqi.com/`
- 注册、游戏大厅、正式对局：`play.xiangqi.com`
- APK、Google Play、App Store：真实下载链接
- 社交媒体：知乎、微信公众号、B 站、小红书、WhatsApp 等源站页脚外链

### 必须本地化的资源

- 图片、背景图、icon、应用商店图标。
- CSS。
- JS。
- 字体。
- SEO 页面正文。
- 教学文章正文。
- FAQ 内容。

### 禁止

- 禁止直接加载源站建站系统脚本。
- 禁止加载源站 CSS。
- 禁止依赖 `original` 懒加载运行时作为最终方案。
- 禁止使用转码损坏的 HTML 作为源码。
- 禁止页面普通导航跳回源站。

## 5. 页面实现要求

### 首页

- 像素级复刻桌面首屏。
- 保留所有 SEO 长内容模块。
- 教学文章区域需有真实卡片和详情跳转。
- FAQ 区域需可展开/收起。
- 语言下拉跳转本地多语言页面。

### 帮助中心

- 保留帮助项目列表。
- 所有帮助子项进入本地复刻页。
- 允许从英文帮助菜单补全简中未暴露但站点存在的子页面。

### 下载页

- 真实 APK / Google Play / App Store 链接允许保留。
- 图标和按钮视觉需与源站一致。
- 下载说明和 FAQ 保留。

### 教程页

- 保留目录、作者信息、长文正文。
- 目录锚点可点击滚动。
- 多语言切换进入对应语言页面。

### 教学文章

- 列表页：文章卡片、日期、摘要、更多入口。
- 详情页：标题、日期、作者、正文、图片、上下篇/返回入口。
- 首页文章区点击进入本地文章详情。

### FAQ

- 首页 FAQ 需要复刻真实交互。
- FAQ 项展开/收起状态可管理。
- FAQ 内部链接指向本地页面。
- 允许业务外链按规则保留。

## 6. 多语言规格

| 语言 | 本地路由 | 来源 |
| --- | --- | --- |
| 简中 | `/` | `www.zh.xiangqi.com` |
| 繁中 | `/tcn/` | `www.zh.xiangqi.com/tcn/` |
| English | `/en/` | `www.xiangqi.com` |
| Tiếng Việt | `/vi/` | `www.vn.xiangqi.com` |

要求：

- 每个语言版本至少复刻首页、帮助、下载、教程核心页。
- 繁中、英语、Tiếng Việt 不允许降级为仅翻译文案或简单入口页；必须依据对应语言源站页面复刻实际页面、实际内容和实际交互。
- 多语言页面的图片、背景图、icon、按钮图标、文章配图、FAQ 内容和 SEO 正文均需要本地化或复用本地资源。
- 顶部语言菜单跳转到对应本地语言路由。
- 文案不能机器硬翻已有中文，应以对应目标站页面为基准。
- 静态资源复用本地资源，必要时按语言补充截图/图片。

## 7. 资源抓取流程

1. 抓取 HTML 原始字节，按响应头或页面 `charset` 解码。
2. 提取以下资源：
   - `<img src>`
   - `original`
   - `data-src`
   - CSS `background-image`
   - 内联 JSON 里的图片路径
   - App Store / Google Play badge 图片
   - favicon / logo
3. 下载到本地资源目录。
4. 建立资源映射表。
5. 页面中改为本地路径。
6. 用构建产物扫描确认无未授权远程资源。

## 8. 交互实现

| 交互 | 实现要求 |
| --- | --- |
| 顶部帮助下拉 | 桌面 hover + click；移动端 click 展开 |
| 语言下拉 | click 展开，选择后进入本地语言页面 |
| 移动导航 | 汉堡按钮展开/收起，点击链接后关闭 |
| FAQ | 点击问题展开/收起答案，支持多个或单个展开需对照源站 |
| 教学文章 | 卡片 hover、点击详情、更多入口 |
| 目录锚点 | 教程页目录点击滚动到章节 |
| 下载按钮 | 保留真实下载外链，打开方式对照源站 |

## 9. 验收标准

### 构建

- `astro build` 通过。
- 无源站 CSS / JS 依赖。
- 无未授权远程图片。
- sitemap 中无 `undefined`。

### 资源

- 允许外链只出现在：
  - `play.xiangqi.com`
  - Google Play / App Store / APK 下载
  - 社交媒体
  - schema.org / sitemap XML 命名空间等标准协议标识
- 其余资源必须本地化。

### 编码

- 源码文件 UTF-8。
- 浏览器页面中文、繁中、英文、越南语正常显示。
- 不允许出现转码乱码。

### 视觉

- 桌面、平板、手机截图与源站逐项对照。
- 首页首屏需像素级接近。
- 移动端不允许文字溢出、按钮重叠、图片错位。

### 交互

- 导航、语言、FAQ、文章、目录、下载入口均可操作。
- 键盘 `Esc` 可关闭菜单类浮层。
- 移动端菜单可用。
