# 相弈象棋像素级复刻：设计规格

版本：v0.3  
日期：2026-04-28

## 1. 设计目标

按像素级复刻目标站 `https://www.zh.xiangqi.com/`，首页优先级最高。复刻不等于重新设计；任何视觉调整都必须服务于还原源站。

## 2. 已确认设计原则

- 首页要求像素级复刻。
- 保留目标站 SEO 长内容。
- 下载图标、App Store / Google Play 图标、社交图标需与源站一致。
- 允许业务外链和社交外链。
- 多语言必须实际做页面。
- 繁中、英语、Tiếng Việt 页面必须按简体中文界面的同等复刻要求实现，包含像素级视觉目标、图片与图标一致性、内容顺序、交互状态和移动端表现。
- 教学文章和常见问题区必须复刻实际交互效果。

## 3. 首页视觉结构

### 顶部导航

必须还原：

- 红色导航栏。
- 左侧 Xiangqi.com 相弈象棋 logo。
- 右侧导航：帮助、下载APP、如何下象棋、简中。
- 帮助和语言有下拉箭头。
- 下拉菜单颜色、间距、hover 状态对照源站。

### 首屏

必须还原：

- 山林雾气背景。
- 左侧悬浮棋子。
- 右侧棋盘。
- 中央标题：`马上进行中国象棋 / 在线对战吧！`
- 两个 CTA：`在线对弈`、`下载 App`
- 按钮尺寸、圆角、红色、间距、文字位置对照源站。

### 红色内容区

必须还原：

- 红色整屏横向区块。
- 大号白色标题。
- 白色正文。
- 对应图片位置和尺寸。

### 后续 SEO 内容

需保留并按源站顺序组织：

- 免费象棋在线玩。
- 双人在线象棋对战。
- 中国象棋游戏在线玩。
- 在线下象棋提升技能。
- 人机对战。
- 残局游戏。
- 在线赛事。
- 排行榜。
- 国际化社群。
- 不同设备。
- 注册说明。
- 礼仪。
- 未来趋势。

## 4. 教学文章区设计与交互

源站首页包含教学文章入口，不可只做静态文字。

必须实现：

- 文章卡片或列表。
- 标题、摘要、日期、图片。
- “更多 / View All”入口。
- 点击文章进入详情页。
- 详情页保留标题、作者、日期、正文、图片。
- 文章卡片 hover 状态按源站复刻。

## 5. FAQ 区设计与交互

必须实现：

- FAQ 问题列表。
- 点击问题展开答案。
- 再次点击可收起。
- 展开图标/箭头状态需跟随变化。
- FAQ 中内部链接指向本地页面；允许业务外链按规则保留。
- 移动端展开后不遮挡后续内容。

## 6. 多语言设计

必须做实际页面：

| 语言 | 本地入口 | 设计要求 |
| --- | --- | --- |
| 简中 | `/` | 对照简中源站 |
| 繁中 | `/tcn/` | 对照繁中源站 |
| English | `/en/` | 对照英文源站 |
| Tiếng Việt | `/vi/` | 对照越南语源站 |

多语言页面统一要求：

- 繁中、英语、Tiếng Việt 不是简中页面的占位翻译版本，而是与简中同等级别的复刻对象。
- 每个语言版本的首页首屏、导航、帮助、下载、教程、教学文章、FAQ、页脚和移动端布局均需按对应源站还原。
- 对应语言的按钮、标题、正文、SEO 长文、文章详情、FAQ 展开文案和图标状态都必须正常显示。

语言菜单：

- 当前语言显示在导航末尾。
- 点击展开语言列表。
- 点击语言进入本地对应语言页面。
- 不直接跳外部语言站。

## 7. 响应式设计

| 断点 | 要求 |
| --- | --- |
| `>= 1440px` | 对照源站桌面截图，重点看首屏比例 |
| `1200px` | 导航横排，首屏图完整 |
| `768px` | 切换移动/平板布局，内容不重叠 |
| `390px` | 手机主验收宽度，导航、按钮、FAQ、文章卡片必须可操作 |

移动端必须验证：

- 汉堡菜单可打开。
- 帮助下拉可展开。
- 语言下拉可切换。
- Hero 标题不遮挡图片主体。
- CTA 不溢出屏幕。
- FAQ 展开后不遮挡内容。
- 文章卡片不横向滚动。

## 10. 像素级复刻注意事项

- 先固定目标截图，再实现页面，避免凭印象调整。
- 页面字体、字号、行高、颜色、图片比例、按钮宽高必须有记录。
- 复杂的源站建站系统 HTML 不直接作为最终源码，但可作为测量参考。
- 复刻时优先还原用户可见效果，而不是还原源站内部 DOM 类名。
- 对于源站动态交互，使用本地等效实现，但视觉和行为必须一致。
