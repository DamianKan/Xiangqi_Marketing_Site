## 3. 首屏专项检查

| 项目 | 通过标准 |
| --- | --- |
| Logo | 与源站尺寸、位置、清晰度一致 |
| 红色导航 | 高度、色值、菜单间距、下拉箭头一致 |
| 左侧悬浮棋子 | 不缺图、不错位、不被裁坏 |
| 右侧棋盘 | 不缺图、不错位、不压住标题 |
| 山林背景 | 裁切和纵向位置接近源站 |
| 标题 | 字体、字号、行距、颜色、位置接近源站 |
| CTA | 宽度、高度、圆角、间距、hover 状态接近源站 |
| 首屏高度 | 红色下一段露出位置与源站接近 |

## 5. 自动化建议

后续新增 `scripts/capture-replica-screenshots.mjs`：

```text
1. 启动本地 Astro preview。
2. 用 Playwright 访问本地路由和源站路由。
3. 按视口保存 before/after 截图到 artifacts/screenshots/。
4. 输出缺图、乱码、控制台错误、外链违规报告。
```

## 6. 失败判定

- 页面出现乱码，直接失败。
- 首页首屏关键图片缺失，直接失败。
- 普通导航跳回源站，直接失败。
- 多语言页面只是简中占位或简单翻译，直接失败。
- 移动端出现横向滚动或内容遮挡，直接失败。
