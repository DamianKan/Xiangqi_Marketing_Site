# 内容维护说明

本项目按“内容文件化优先”组织，方便运营团队在不改页面模板的情况下维护站点内容。

## 常用文件

- `src/data/pages.json`：页面标题、描述、摘要块、路径与 H1。
- `src/data/faqs.json`：FAQ 问答内容。
- `src/data/articles.json`：新闻与文章内容。
- `src/data/navigation.json`：导航配置。

## 推荐修改流程

1. 仅改文案或 SEO 信息时，优先修改上述 JSON 文件。
2. 本地启动后检查页面是否正常显示。
3. 提交分支并创建合并请求。
4. 通过预览环境确认效果后再合并主分支。

## 什么时候需要改代码

- 页面结构变化。
- 新增页面类型。
- 样式体系调整。
- 新增结构化数据模板或新端点。
